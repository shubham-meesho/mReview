## Past Incident Patterns
- `userId % rolloutValue == 0` false-positive enablement: Using `0` as a "disabled" rollout percentage in configs caused the modulo hash to evaluate as enabled for users whose hash was divisible by zero, leading to unintended traffic exposure; team now mandates `-1` as the canonical off-value.
- VSKU replacement leaking into unintended flows: The VSKU experiment logic in `BrowsingHistoryHandler` (around line 190) replaces original products, a behavior that was known to be inactive/unsafe for wishlist-style flows; reusing wishlist handler code silently carried this side effect into Browsing History, risking catalog data corruption for that surface.
- Config type mismatch causing silent misloads: The `threshold` field was loaded as a `float` but declared as a non-float type in config files, causing silent type coercion issues in `application-dyn-prd.yml` and `application-dyn-int.yml`; the fix required explicitly typing the field as a string in YAML and updating the schema.
- Comment/code version drift in `EnhancementDataAdapter`: The iOS version gate was changed from `>= 100` to `>= 255` in code but the comment retained the old value, masking the actual behavior boundary during review and risking incorrect future edits based on stale documentation.
